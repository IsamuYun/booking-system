# Redesign brief

目标：
基于当前既有项目重新设计和开发 UI，不是新建孤立原型。

请先阅读：
- 项目 README / package.json / src 目录
- docs/redesign-reference/open-design/README.md
- docs/redesign-reference/open-design/SKILL.md
- docs/redesign-reference/open-design/colors_and_type.css
- docs/redesign-reference/open-design/ui_kits/miniprogram/README.md
- docs/redesign-reference/open-design/ui_kits/miniprogram/*.jsx
- docs/redesign-reference/open-design/ui_kits/miniprogram/kit.css

要求：
1. 先理解既有项目的技术栈、路由、组件结构。
2. 再提取 Open Design 参考里的颜色、字体、间距、组件风格。
3. 不要直接覆盖整个项目。
4. 先给出改造计划：哪些页面、哪些组件、哪些 CSS/token 文件需要改。
5. 然后按小步提交式修改。
6. 保留既有业务逻辑，只重构 UI、布局、交互和样式。
